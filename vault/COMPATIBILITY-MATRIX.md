## Key findings

1. **Producers vs. server processors.** Only client-only producers (Vault Notes, Foundry Collector, most CLI utilities) can be legitimately zero-knowledge vs. Vault infra. Every hosted service (Gateway, loop-mcp, microloan, SpeedSpend) necessarily processes plaintext and must advertise `zero_knowledge:false` / `server_sees:"plaintext_input"`. The contract encodes this line (§2, §7) so no app can falsely claim zero-knowledge.
2. **Evidence-bearing products compound; commodity utilities do not.** Foundry Collector, Settlement Probe, and the Audit produce *signed, scarce* artifacts whose value grows as more accumulate. Utilities (satsledger, bolt12-tools, AgentReady) are easy but low-compounding.
3. **The capsule shape already fits the DB.** The exporter's output maps 1:1 onto live `lv_capsules` columns — no schema migration needed to list a Foundry capsule.

**Decision:** Foundry Collector is the first integration — lowest difficulty, highest compounding value, genuinely zero-knowledge, and its AOC is already a signed owner-retained artifact one step from a Vault capsule.
